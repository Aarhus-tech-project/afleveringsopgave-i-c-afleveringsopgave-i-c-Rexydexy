# 💻 Afleveringsopgave i C#
 
**Datatekniker · Hovedforløb 1 · Grundlæggende Programmering (16471)**
 
---
 
## 🎯 Formål
 
Du skal udvikle et mindre **konsolprogram i C#**. Rammen er fast – emne og indhold vælger du selv. Det forventes, at du anvender og dokumenterer fagets målpinde i din løsning. Du afleverer **kun kildekoden** via dette repository.
 
## 🧱 Rammen (faste krav)
 
- ⚙️ Konsolapplikation i C#, der kan kompileres og køres uden fejl.
- 📋 En menu/brugerinteraktion med mindst to handlinger at vælge imellem.
- 🗂️ Behandler data ved hjælp af **arrays** (f.eks. liste, bræt, lager, pointtavle).
- 🧩 Opdelt i flere **metoder** – ikke al logik i `Main`.
- 🛡️ Håndterer forkert brugerinput fornuftigt.
- 🔀 Versionsstyret i Git med løbende commits gennem hele forløbet.
## 💡 Emnet vælger du selv (inspiration)
 
Vælg noget, der lader dig ramme målpindene:
 
- 🎮 Tekstbaseret spil (gættespil, terningespil, mini-quiz, kryds og bolle på et bræt)
- 📇 Registreringssystem (medlemsliste, lagerstyring, karakterregister, opgaveliste)
- 🔢 Lommeregner eller omregner med flere funktioner
- 🏭 Simpel simulering (kassesystem, parkeringshus, automat)
## ✅ Vurdering – der lægges vægt på
 
| Område | Hvad vi kigger efter |
|---|---|
| 🚀 Funktionalitet | Programmet løser opgaven og kører uden fejl |
| 📖 Kodekvalitet | Læsbar kode med fornuftige navne, indrykning og relevante kommentarer |
| 🔡 Datatyper & variable | Bevidst valg af datatyper samt brug af variable og konstanter |
| 🔀 Kontrolstrukturer | Brug af flere forskellige (`if/else`, `switch`, `while`, `do-while`, `for`, `foreach`) |
| 🧩 Metoder | Opdeling i metoder med og uden returværdi og parametre |
| 🗂️ Arrays | Brug af både enkelt- og multidimensionelle arrays |
| 🔁 Versionsstyring | Løbende commits i Git med sigende beskeder |
| 🐞 Test & debugging | Dokumenteret fejlfinding med debugger og en kort test |
 
## 🎓 Målpinde der skal anvendes
 
Det forventes, at du anvender målpindene fra faget. Sørg for, at hver enkelt målpind kan genfindes et sted i din løsning.
